面试题测试

1. 如果AB两个系统互相依赖，如何解除依赖

考虑相互依赖的两个要素：

  1. 依赖方向性：从业务架构视角，考虑业务流程的优化

    【案例1】平台订单重构改造，去掉order_engine，业务单向依赖平台订单

    【案例2】乐卡静态权益，下单流程中，乐卡侧先扣减权益配额，再到电商侧使用优惠

  2. 是否循环依赖：梳理系统的整体依赖关系，按业务模块/功能分层，限定层次间单向依赖

    【案例1】核心平台交易底层服务下沉

2. 什么场景应该拆分系统，什么场景应该合并系统

3. 什么是服务降级，并举例说明

由于系统资源不足，或者服务内部错误，导致服务不可用时，为了缓解影响用户体验，使用降级方案。

【案例1】系统资源不足：双十一期间，支付宝暂停还款功能；

【案例2】服务内部错误：乐黑卡查询失败，默认当乐卡用户处理；

与服务熔断区分：所依赖的下游服务不可用（超时频繁或者报错），为避免引发雪崩，临时断开对问题服务的请求。一般熔断都有对应降级策略。

4. 介绍3个常用的设计模式，及其应用场景

设计模式可以分为三大类：创建型、结构型、行为型，对应的介绍一种常用类型

  1. 工厂方法

    • Product：抽象产品类
    • ConcreteProduct：具体产品类
    • Factory：抽象工厂类
    • ConcreteFactory：具体工厂类

  2. 代理模式

    • Subject: 抽象主题角色
    • Proxy: 代理主题角色
    • RealSubject: 具体主题角色

  3. 模板方法模式

    • AbstractClass 抽象类
    • AbstractClass#templateMethod 模板方法
    • ConcreteClass 具体子类

5. 常见的缓存有哪些，如何使用

缓存可以分为两类：本地缓存、分布式缓存

常见的本地缓存有：静态变量（一般是 HashMap）、Guava Cache

常见的分布式缓存有：Memcache、Redis

关于缓存使用，可以分为以下几种场景：

  1. 预加载数据：更新频率低&查询频繁的数据，按是否强一致性选择分布式或者本地缓存

    【分布式缓存】秒杀系统中的商品数据、DB配置等应用配置信息

    【本地缓存】系统运营性配置（如文案展示等）

  2. 预分配资源：有限的资源，访问频繁导致争抢严重，需要加锁的场景

    【分布式缓存】秒杀商品库存扣减利用redis原子扣减特性、领券场景预扣部分库存放redis

    【本地缓存】序列号生成系统，预申请号段缓存本地

  3. 分布式锁
